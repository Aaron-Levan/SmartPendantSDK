Stations

Stations form the foundation of a palletization cell.  They are the zones of the palletization cell where products can be picked or placed. 

Every station has a user frame associated with it.  User frames are taught using the Smart Pendant user frames menu which is accessible by pressing the gear icon next to the "User Frame" selection box. 

When teaching points, the taught points will be with respect to the assigned user frame.  For proper operation, it is important that the user frame be taught at the recommended location for each station type.  For details on how to teach the user frame for a specific station type, press the information icon next to the "User Frame" dropdown list. 

In the variable assignment section, users must assign variables that the palletization jobs will reference to for that station.  Following variable assignment, I/O can be selected to add further control to the station.  These I/O can be used to signify to a controller when a station is ready, needs resetting, or a pallet stack is complete. 

Optionally, users can also assign jobs that will be called when transitioning between stations.  These jobs can be used to move the robot along a specific path between stations to avoid obstacles such as fencing, a column or other stations.  They can also be used to activate I/O signals to the gripper or other devices. 

Station Specific:

Build Station:

  • Pattern S-Var: String variable used to store the assigned Build Pattern job name. 
  • Build Height D-Var: Double integer variable used to track the current height of the build station (in microns) and shift the layer accordingly. 
  • Layer I-Var: Integer variable with the build current layer number (pallet and separators also count as layers). 
  • Cycle I-Var: Integer variable with the layer current cycle (box) number. 
  • Ready Input#: Input signal confirming that the build station is ready.  Often associated with a pallet present sensor. 
  • At Station Output#: Output signal indicating that the robot is currently at this station. 
  • Reset Input#: Input signal causing the build station to reset the build height to 0 and the layer and cycle variables to 1. 
  • Complete Output#: Output signal indicating that the current build is complete. 

Note: The Complete Output and Reset Input usually work together. When the build pattern is complete, the Complete Output turns on, once the pallet is removed and a new one is in position, the Reset Input is turned on. When the Complete signal turns off, the Reset Input can be turned off.

Infeed Station:

  • Alignment: Indicates if the boxes on the infeed are aligned to the right, left or center of the conveyer. 
  • Width: Width of the conveyer in mm. Used to calculate the position of the box relative to the user frame. 
  • Product S-Var: String variable used to store the product name assigned to the infeed. 
  • Ready Input#: Input signal confirming that the infeed station is ready.  Often associated with a box present sensor at the end of the infeed. 
  • At Station Output#: Output signal indicating that the robot is currently at this station. 

Dispenser Fixed Pick:

  • Product S-Var: String variable used to store the product name assigned to the dispenser. 
  • Ready Input#: Input signal confirming that the dispenser station is ready. Often associated with a sensor detecting the presence of product. 
  • At Station Output#: Output signal indicating that the robot is currently at this station. 

Dispenser Search Stack:

  • Height: Height of the dispenser sides that need to be cleared when moving to or from the station. 
  • Product S-Var: String variable used to store the product name assigned to the dispenser. 
  • Search Adjustment D-var: Double variable used to track the location of the last found product.  If enabled, the next search operation will start from the last found location, instead of starting the search from the maximum stack height. 
  • Ready Input#: Input signal confirming that the dispenser station is ready.  Often associated with a sensor detecting the presence of product. 
  • At Station Output#: Output signal indicating that the robot is currently at this station. 
  • Reset Input#: Input signal causing the dispenser to reset empty stack output and the search adjustment height to 0. 
  • Empty Stack Output#: Output signal indicating that the search has reached the taught pick position on the last search. 
  • Search Input#: Input signal from the gripper detecting the presence of the product.  There is an option to ‘Activate tool at beginning of search’, this allows to turn on vacuum at the beginning of the search when a vacuum confirmation sensor is used as the search input. 
  • When the product is thin and the ‘Search Adjust’ variable is enabled, a distance can be added to ‘Start the next search Z mm above the last found part’. 
  • Signal Low Stack Output: When enabled, the indicated output signal will turn on when the found product is below the indicated value in mm. 

Note: The Empty Stack Output and Reset Input usually work together.  When the dispenser is empty, the Empty signal turns on, once the dispenser is reloaded, the Reset Input is turned on.  When the Complete signal turns off, the Reset Input can be turned off.
